===== 3. CONTEXTO DA REGRA =====
A auditoria PTP 113 C.8.2.4 testou endpoints em http://127.0.0.1:8766.
O serviço real informado pelo Leo estava em http://127.0.0.1:5001/mobile.
A validação posterior PTP 113 C.8.2.4.2 confirmou PASS_VALIDATION na porta 5001.

Conclusão:
- 8766 foi premissa incorreta por histórico.
- 5001 era a porta real ativa do Mobile V2 naquele momento.
- Falha de conexão em porta não confirmada deve ser classificada como SERVER_NOT_RUNNING ou PORTA_NAO_CONFIRMADA.
- Não deve ser classificada automaticamente como falha do app.

===== 4. REGRA OFICIAL REGISTRADA =====
REGRA — Auditoria HTTP PredixAI BR

Nunca assumir porta pelo histórico.

Antes de qualquer validação HTTP:
1. detectar porta ativa quando possível;
2. aceitar BASE_URL configurável;
3. registrar BASE_URL no relatório;
4. testar a porta real informada pelo Leo;
5. se houver dúvida, testar portas candidatas, como 5001 e 8766;
6. se nenhuma porta responder, classificar como SERVER_NOT_RUNNING ou PORTA_NAO_CONFIRMADA;
7. não classificar como falha do app sem servidor confirmado;
8. diferenciar:
   - conexão recusada/HTTP 000 = porta/servidor não confirmado;
   - HTTP 4xx = rota, método ou contrato possivelmente incorreto;
   - HTTP 5xx = possível falha do app;
   - HTTP 200/201 = endpoint saudável.
