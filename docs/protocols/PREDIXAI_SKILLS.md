# SKILLS E COMANDOS CURTOS — PREDIXAI BR

| Comando | Função |
|---|---|
| `saúde` | Mostra saúde do chat. |
| `iniciar` | Executa protocolo de início. |
| `fechar` | Gera checkpoint histórico final. |
| `checkpoint` | Alias de `fechar`. |
| `roadmap` | Mostra roadmap visual. |
| `repo` | Prepara auditoria do repositório. |
| `script` | Gera script all-in-one após debate e aprovação. |
| `validar` | Valida etapa atual. |
| `mobile` | Valida fluxo mobile-first. |
| `mini` | Cria mini-PTP. |
| `md` | Gera/atualiza Markdown histórico. |
| `estado` | Mostra estado atual. |
| `riscos` | Lista riscos. |
| `relatório` | Interpreta relatório colado. |

## Regras

1. Comandos devem ser curtos.
2. Não criar comandos parecidos para a mesma função.
3. Não executar script sem plano aprovado.
4. Não avançar etapa sem relatório analisado.
5. Não misturar outra PTP/letra principal no mesmo chat.
