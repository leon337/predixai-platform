# PTP 113 C.7.4 — Corrigir Lock SQLite do Reader

## 1. Identificação

- PTP: PTP 113 C.7.4
- Nome curto: corrigir_lock_sqlite_reader
- Data: 2026-07-07 11:29:13 -0300
- Status: FIX
- Relatório TXT: `reports/20260707_112913_PTP-113C.7.4_corrigir_lock_sqlite_reader_FIX.txt`

## 2. Objetivo

Corrigir a queda do reader visual causada por `database is locked` durante leitura real observada da corretora.

## 3. Contexto

Na PTP 113 C.7.3, o reader confirmou captura real:

```txt
LATAM Index detectado
preço capturado
current_signal_status=sinal_pronto
operation_allowed=true
```

A falha passou a ser concorrência SQLite entre servidor mobile, reader e múltiplas chamadas para `/api/mobile/state`.

## 4. Correção aplicada

Foi adicionada camada de resiliência SQLite:

```txt
PRAGMA busy_timeout=30000
PRAGMA journal_mode=WAL
PRAGMA synchronous=NORMAL
retry curto em database is locked
```
