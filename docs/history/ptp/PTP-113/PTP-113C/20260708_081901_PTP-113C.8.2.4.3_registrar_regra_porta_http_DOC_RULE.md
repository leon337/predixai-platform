## 2. Objetivo

Registrar formalmente que auditorias HTTP do PredixAI BR não devem assumir porta pelo histórico do projeto.

## 3. Contexto

A auditoria `PTP 113 C.8.2.4` testou endpoints na porta `8766` e retornou WARN por conexão recusada.

Leo informou que o serviço real estava em:

```txt
http://127.0.0.1:5001/mobile
```

A etapa `PTP 113 C.8.2.4.2` validou a porta correta `5001` com PASS_VALIDATION.

`PASS_DOC_RULE`

## 8. Decisões

Regra oficial:

```txt
Nunca assumir porta pelo histórico.
Antes de qualquer validação HTTP:
1. detectar porta ativa quando possível;
2. aceitar BASE_URL configurável;
3. registrar BASE_URL no relatório;
4. testar a porta real informada pelo Leo;
5. se houver dúvida, testar portas candidatas, como 5001 e 8766;
6. se nenhuma porta responder, classificar como SERVER_NOT_RUNNING ou PORTA_NAO_CONFIRMADA;
7. não classificar como falha do app sem servidor confirmado.
```

## 9. Pendências

- Aplicar esta regra em todo script HTTP futuro.
- Reabrir C.8.3 somente após plano aprovado.

## 10. Próxima ação

Planejar a nova C.8.3 usando porta confirmada/detectada e validação HTTP segura.
