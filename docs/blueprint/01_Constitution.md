# PredixAI BR — Constituição v1.0

## 1. Missão

Criar uma plataforma de inteligência artificial capaz de observar, interpretar, auditar e automatizar processos digitais com segurança, transparência e escalabilidade.

## 2. Visão

Transformar a PredixAI BR em uma plataforma modular de IA aplicada, onde diferentes produtos possam nascer sobre o mesmo núcleo tecnológico.

## 3. Primeiro produto

O primeiro produto será o PredixAI Trader.

Ele será usado para validar visão computacional, auditoria, análise de dados e estratégia assistida por IA.

## 4. Princípios

### Segurança primeiro

Nenhuma funcionalidade deve comprometer dados, licença, privacidade ou integridade do sistema.

### Transparência

Toda decisão do sistema deve ser explicável.

### Modularidade

Cada módulo deve ter responsabilidade clara.

### Escalabilidade

Toda decisão deve considerar crescimento futuro.

### Auditoria total

Tudo que o sistema observar ou decidir deve ser registrado.

### IA como analista

A IA ajuda a interpretar dados, mas não assume controle financeiro na V1.

## 5. Papéis

### Leo

CEO e operador humano da PredixAI BR.

Responsável por:

- validar ideias;
- testar o produto;
- criar contas;
- executar comandos;
- revisar usabilidade;
- alimentar redes sociais;
- tomar decisões finais.

### ChatGPT

CTO, arquiteto e diretor de produto.

Responsável por:

- arquitetura;
- roadmap;
- requisitos;
- revisão técnica;
- design conceitual;
- segurança;
- organização do projeto;
- decisões estratégicas.

### Codex

Desenvolvedor oficial.

Responsável por:

- criar arquivos;
- implementar módulos;
- corrigir bugs;
- criar testes;
- documentar código;
- atualizar contexto do projeto.

## 6. Regra de ouro

O Codex não decide arquitetura sem aprovação.

## 7. Regra de versionamento

Toda mudança relevante deve atualizar:

- predixai_context.json
- CHANGELOG.md
- documentação relacionada

## 8. Cultura

A PredixAI BR será desenvolvida como produto comercial desde a primeira linha de código.
